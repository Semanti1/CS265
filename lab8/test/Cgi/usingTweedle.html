Using tweedledee.cs.drexel.edu

    Intro

    For security reasons we have set up a box, tweedledee, for us to run scripts on (everybody thank Cosmin for all of this work). Our home directories are not mounted, but we have the same login and password.

    Again, for security purposes, we don't allow connections to go *out* from tweedledee. This means that you'll need to push and pull files into/out of tweedledee from one of the other department machines, using scp or maybe sftp

    Warning: You don't want to put any sensitive data out on this server. You will also want to back up your work to your regualr CS home directory.

    Directories

    You'll have the normal public_html directory. You should give the word read & execute access, if you want to access the pages from the Web.

    In your home directory is also a link called cgi to a directory elsewhere on the machine. Just treat it like a directory. This is where your CGI "scripts" will go. Any scripts will need to give the world read & execute permission.

    Requesting Pages from Outside

    To get to a webpage:

    http://tweedledee.cs.drexel.edu/~userId/page.html

    To request a script:

    http://tweedledee.cs.drexel.edu/cgi-bin/userId/script

    Copying Files into/out of Tweedledee

    Log into one of the department machines, as you normally would. Make a place somewhere to keep your files for this lab.

    Using scp to place files onto Tweedle:

    scp hello.html tweedledee:public_html/
    scp hello.cgi tweedledee:cgi/

    using scp to retrieve files from Tweedle:

    scp tweedledee:public_html/hello.html .
    scp tweedledee:cgi/hello.cgi .

    Referring to Scripts From Within a Webpage

    The path, from within the server, is: /cgi-bin/userId/script. E.g.:

    <form action='/cgi-bin/kschmidt/hello.pl' method='post' enctype='multipart/form-data'>

    Please note, this setup is specific for running in the sandbox here on Drexel's CS Dept.'s servers. The coding, however, is not.
